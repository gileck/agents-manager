# Screenshot Capture Guide

Place 3–5 screenshots of key app pages here. These give the UX Designer agent
a visual density / whitespace / "feel" reference that CSS tokens alone cannot convey.

## Required screenshots

| File name              | What to capture                                              |
|------------------------|--------------------------------------------------------------|
| `dashboard.png`        | Main dashboard / task list view with sidebar visible         |
| `task-detail.png`      | A single task detail page (overview tab)                     |
| `agent-conversation.png` | Agent chat / conversation panel showing agent blocks       |
| `review-diff.png`      | Review page with diff panel open                             |
| `settings.png`         | Settings or configuration page                               |

## How to capture

1. Run the app in **dark mode** (the primary theme).
2. Use a **1440 × 900** browser / Electron window for consistent framing.
3. Use the OS screenshot tool or DevTools "Capture screenshot" to get a clean PNG.
4. Name files exactly as listed above so the UX Designer agent can reference them.

## Notes

- Prefer dark mode screenshots (the app defaults to dark).
- Crop to the app window only — no OS chrome / menu bar.
- These are reference material, not pixel-perfect specs.
